### [NATO — Neural Adaptive Training Optimizer](https://github.com/Malhar1912/NATO)

[![PyPI](https://img.shields.io/pypi/v/nato-opt?style=flat-square&labelColor=0d0d1f&color=2e2a52&logo=pypi&logoColor=c4cfe0)](https://pypi.org/project/nato-opt/)
![](https://img.shields.io/badge/python-3.8+-1a1a3e?style=flat-square)
![](https://img.shields.io/badge/CUDA-GPU%20accelerated-0d1a0d?style=flat-square&logo=nvidia&logoColor=76b900)
![](https://img.shields.io/badge/license-MIT-2e2a52?style=flat-square)
![](https://img.shields.io/badge/co--authored-Atharva%20Khambete-2e2a52?style=flat-square)

A bespoke PyTorch optimizer of considerable audacity, published to PyPI and installable by anyone with the requisite discernment. Co-authored with Atharva Khambete. Three original regularization mechanisms, each operating atop existing forward passes — contributing precisely zero additional computational overhead, which is the kind of elegance one simply does not encounter enough of.

```
pip install nato-opt
```

**The triptych of innovations:**

| Mechanism | Modus Operandi |
|---|---|
| **Fourier Spectral Penalty (FSP)** | Imposes regularization upon model weights in the *frequency domain* — not the weight domain, which would be pedestrian — penalising high-frequency spectral components with surgical disregard for their feelings |
| **Kakeya Directional Penalty** | Penalises gradient directions exhibiting excessive *consistency* — because certainty, left unchecked, is merely collapse wearing a confident expression |
| **N-D FFT Gradient Filtering** | Applies an N-dimensional low-pass filter to the gradient signal prior to each optimizer step, smoothing the high-frequency noise that accumulates like unread correspondence |

```python
from nato_opt import NATOOptimizer, fourier_spectral_penalty, low_pass_filter_gradients

optimizer = NATOOptimizer(model.parameters(), lr=1e-3)
fsp       = fourier_spectral_penalty(model, lambda_fsp=1e-6)
low_pass_filter_gradients(model, cutoff_ratio=0.5)
optimizer.step(epoch)
```

### [GeoSDE-IDS — Geometric Stochastic Differential Intrusion Detection System](https://github.com/Malhar1912/GeoSDE-IDS)

![](https://img.shields.io/badge/Elsevier-manuscript%20submitted-2e2a52?style=flat-square&logo=latex&logoColor=c4cfe0)
![](https://img.shields.io/badge/CIC--IDS%202017%20%7C%20NSL--KDD%20%7C%20UNSW--NB15-3%20datasets-1a1a3e?style=flat-square)
![](https://img.shields.io/badge/5%20seeds-reproducible-2e2a52?style=flat-square)
![](https://img.shields.io/badge/Python%20%7C%20TeX%20%7C%20Neural%20SDEs-hyperbolic%20geometry-1a1a3e?style=flat-square)

A research-grade Intrusion Detection System of extraordinary geometrical ambition. Where lesser systems deploy thresholds and heuristics, GeoSDE-IDS abandons Euclidean space *entirely*, electing instead to model network traffic dynamics upon **hyperbolic manifolds** — specifically the Poincaré Ball and Lorentz Hyperboloid — exploiting the hierarchical topology of network graphs with the confidence of someone who has read all the relevant differential geometry literature and then written their own.

Anomaly detection occurs not through the violation of numerical bounds but through the *distortion of curvature* — the manifold itself signals that something is profoundly wrong.

**The quintet of detection signals, running in ruthless parallel:**

```
  ① Ollivier-Ricci Curvature Distortion      — the geometry of local neighbourhoods, disturbed
  ② Dirichlet Energy Functional Acceleration  — the rate of change of spatial smoothness
  ③ Möbius Invariance Violation               — symmetry that ought to hold, broken
  ④ Fokker-Planck Density Evolution           — probabilistic drift, measured via KL-Divergence
  ⑤ SDE Drift Deviation                       — departure from the learned stochastic trajectory
```

Signals are fused via meta-classifier (Logistic Regression + MLP). Full LaTeX manuscript, benchmarking CSVs, and publication-quality figure generation committed to the repository. Reproducibility checklist observed with characteristic rigour.

### [TCR — Threshold-Constrained Replay: A Constrained Multi-Objective Replay Framework](https://github.com/Malhar1912/Threshold-Constrained-Replay-TCR-A-Constrained-Multi-Objective-Replay-Framework)

![](https://img.shields.io/badge/reinforcement%20learning-constrained%20optimization-2e2a52?style=flat-square)
![](https://img.shields.io/badge/Atari--100k%20%7C%20DMC%20%7C%20Procgen-benchmarks-1a1a3e?style=flat-square)
![](https://img.shields.io/badge/7%20phases%20%7C%20formal%20proofs%20%7C%20ablation%20gates-in%20progress-2e2a52?style=flat-square)

A formal reimagination of experience replay in reinforcement learning — executed across seven rigorously documented phases with the kind of methodological thoroughness that borders on the punishing. Prioritized Experience Replay treats sampling as a ranking problem. TCR treats it as what it actually is: a **constrained stochastic resource allocation problem**, with a formal objective one is obliged to satisfy before proceeding further.

$$\max_{q \in \Delta}\ \mathbb{E}_{\tau \sim q}\bigl[\Delta\mathcal{L}(\tau)\bigr] \qquad \text{subject to} \qquad \mathrm{KL}(q\,\|\,p) \le \varepsilon, \quad |D^*| \le B$$

Phase 0.4 functions as a mandatory validation gate — the proxy approximation validity experiment — which must achieve a Pearson correlation ρ > 0.3 at p < 0.05 across no fewer than 80% of environments before any subsequent phase is permitted to commence. Phases 1 through 7 are not a suggestion. The framework benchmarks against PER, DreamerV3 native, and Uniform Replay across Atari-100k, DeepMind Control, and Procgen with statistical rigour enforced via IQM and bootstrap confidence intervals.
